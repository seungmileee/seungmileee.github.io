browser 구조 분석

window : 브라우저에서 열려있는 전체적인 창

document : window에서 표시되면 화면

navigator : 브라우저 전체에 대한 내용이 담겨있는 유용한 object

페이지를 브라우저에서 열게 되면 가장 먼저 window라는 오브젝트가 있고,그 안에 DOM , BOM (브라우저에 관련된 아이들 [네비게이터, 패치,,,]), javascript(우리가 작성한 JS)가 들어있다.

-> 그래서 브라우저 위에서 동작하는 것들 만들 때는 dom, bom에 관련된 것들 사용할 수 있는 것이다.

브라우저 콘솔에서 'console.log(window)' 를 해보면 window에 대한 다양한 오브젝트들을 (다양한 API들, document, fetch, history, indexDB ...) 확인해볼 수 있다.


여기서 짚어야 할 중요한포인트는 브라우저에서 window는 글로벌 프로젝트라는 것이다.

그래서 콘솔에서 'console.log(this)' 라고 출력해보면 window가 나오며, innerWidth라고 입력해도 콘솔창에 window.innerWidth가 출력이 된다.


window의 navigator 오브젝트에서는 유용하게 많이 쓰이는 app code name, app name, app version, bluetooth, connection, language, keyboard 등등 확인해볼 수 있다.

또 window자체에서도 우리가 많이 쓰는 사이즈, 사이즈 변화 유무, 스크롤링 처리, 페이지 로드 여부 확인 등등을 확인해볼 수 있다.

window 사이즈

window.screen : 모니터 사이즈

window.outer : 페이지 넘어선 브라우저 전체 사이즈, 브라우저에 있는 url, tab 포함한 전체적인 브라우저의 사이즈 (브라우저 자체의 크기)

window. inner : 우리가 보는 웹 페이지 뿐만 아니라 스크롤바 있다면 스크롤바가 포함된 화면들 합한 사이즈

document.element.clientWidth : 스크롤바 제외한 전체 화면 영역 (스크롤바 제외한 순수 document 페이지의 사이즈 )

property role
window.screen 모니터 사이즈
window.outer 페이지를 넘어선 브라우저 전체 사이즈
window.inner 브라우저 페이지가 표시되는 것
+ 스크롤바 존재할 경우 스크롤바 포함된 화면 합한 사이즈
document.doclement.clientWidth 스크롤바 제외한 순수 document 페이지의 사이즈

브라우저 좌표

1. 브라우저의 좌표

기준: 브라우저의 왼쪽 상단에서 시작

X축은 수평축, Y축은 수직축

X는 오른쪽으로 갈 수록 숫자가 커지며, Y는 밑으로 내려가면 내려갈수록 숫자가 커진다.

2. Element.getBouncingRect()

엘리먼트의 크기와 뷰포트에 상대적인 위치 정보를 제공하는 DOMRect 객체를 반환해주는 메서드

출처 : mdn (https://developer.mozilla.org/ko/docs/Web/API/Element/getBoundingClientRect)

Element.getBouncingClientRect() 를 통해서 top (브라우저 좌표의 X축), left (브라우저 좌표의 Y축), height, width, 요소의 너비, 높이 등과 같은 요소의 사이즈, 위치에 관련된 다양한 정보들을 얻어낼 수 있다.


3. client x,y VS page x,y

사용자가 클릭했을 때 클릭 이벤트의 x,y와 page의 x,y값이 다르다.

사용자가 클릭을 하면 이벤트라는 오브젝트가 우리가 등록한 리스너에 등록이 되는데, 이 이벤트에 client x,y라는 값이 있다.

이는 사용자가 보는 페이지에 상관없이 브라우저에서, 즉 브라우저의 윈도우 창에서 클릭한 지점까지 x,y가 얼마나 떨어져 있는지의 좌표를 제공한다.

이와 달리 page의 x,y는 페이지 자체에의 좌표를 말한다. 그래서 우리 눈에 보이지 않는 페이지의 최초 지점부터 시작해서 클릭한 지점까지의 좌표값을 제공한다.

간단히 정리해보면 다음과 같다

client x,y -> 현재 윈도우 창 -> 클릭한 지점까지의 좌표

page x, y -> 페이지 최초 시작 지점 -> 클릭한 지점까지의 좌표


4. JS에서의 bottom ,right VS CSS에서의 bottom ,right

JS에서의 bottom, right

bottom -> 브라우저 상위에서 요소의 하단까지 얼마나 떨어져 있는지

right -> 브라우저 x축에서 요소의 오른쪽까지 얼마나 떨어져 있는지

(x축과 y축 좌표로부터 계산)

CSS에서의 bottom, right

bottom: 브라우저의 가장 밑단으로부터 요소의 하단까지 얼마나 떨어져 있는지

right: 브라우저의 가장 오른쪽 부분으로부터 요소의 오른쪽 부분까지 얼마나 떨어져 있는지

(브라우저 오른쪽 끝, 밑단으로부터 계산)
