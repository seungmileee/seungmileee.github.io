API

API(Application Programming Interfaces)

  • 간단하게 말해, 프로그램들이 서로 상호작용 하는 것을 도와주는 매개체!
  • 개발자 입장에서는 복잡한 기능을 쉽게 개발할 수있도록 프로그래밍 언어로 코드를 제공해주고,
  • 사용자 입장에서는 내부에 코드가 어떻게 생겼는지 신경쓰지 않아도 쉽게 기능을 가져다 쓸 수 있게 도와준다!

    역할

      1. 서버와 데이터베이스에 대한 출입구 역할을 한다.
      1. 애플리케이션과 기기가 원활하게 통신할 수 있도록 한다.
      1. 모든 접속을 표준화한다.

예를 들어, API를 사용하는 사람은 어떻게 로그인, 로그아웃 작성되었는지 걱정할 필요 없이 여기서 제공하는 함수 이용해 간단하게 로그인, 로그아웃이 가능하다.

Web API

이 API 중, 우리가 지금부터 공부해 볼 WEB API는 개발자가 브라우저 상에서 쉽게 개발할 수 있도록 도와주는 객체 모음들을 말한다.

많이 사용해보았던 DOM API로 시작해, Network API, Graphics API, Audio/Video API, Storage API 등등도 다 웹 API의 한 종류이다.

이 웹 API가 하는 일은 다음과 같다.

  1. 브라우저 로드된 문서를 조작하는 데 도움을 준다.
  2. 웹 페이지의 일부를 업데이트 하는 데 도움을 준다.
  3. 그래픽 요소들을 웹 페이지에 그리는 데 도움을 준다.
  4. 오디오와 비디오 작업을 웹 페이지에서 가능하도록 해준다.
  5. 클라이언트 측 저장소(ex)local storage)를 사용할 수 있도록 도와준다.
